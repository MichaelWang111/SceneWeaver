# Creative Fingerprint 设计方案

## Summary

不使用一个 LLM 请求同时生成 `fingerprint + film_analysis + experience_cards`。采用独立中间层：

```text
scene_analysis
→ scene_fingerprints
→ film_fingerprint
→ film_analysis
→ experience_cards with fingerprints
→ query_fingerprint
→ runtime retrieval
```

理由：`fingerprint` 是检索索引层，`film_analysis` 是解释层，`experience_cards` 是可复用知识层。三者可以共享输入和互相约束，但不应在同一次 LLM 输出里混在一起。

## Key Changes

- 新增 `CreativeFingerprint` schema，作为半受控低维语义坐标。
- 固定 fingerprint 维度，标签值允许 LLM 生成，但必须归一化并保留 evidence。
- 给三类对象挂 fingerprint：
  - scene fingerprint：来自单个 `SceneAnalysis`
  - film fingerprint：由所有 scene fingerprints 按时间聚合
  - card fingerprint：挂在每张 `ExperienceCard` 上，用于 retrieval
- `associate` 增加 query fingerprint：
  - 输入 brief / keywords
  - 输出 `query_fingerprint`
  - 用它去匹配 `experience_cards`，而不是只靠自由联想文本

## Fingerprint Shape

第一版建议字段：

```yaml
emotion_core: []
audience_projection: []
narrative_function: []
interaction_mode: []
visual_motifs: []
symbolic_logic: []
rhythm_pattern: []
evidence: []
confidence: 0.0
```

约束：

- 每个 tag 都要有 evidence，指回 scene id 和原始 reasoning 字段。
- tag 用英文 snake_case，便于匹配和后续 embedding。
- `evidence` 保留中文解释，便于人工检查。
- 不把 fingerprint 写成长段文案，它只做低维索引。

## Implementation Flow

1. `scene_analysis -> scene_fingerprint`
   - 输入：单个 `analysis/scene_XXX.json`
   - 输出：`fingerprints/scene_XXX.json`
   - 目标：抽取低维标签，不重新解释 scene

2. `scene_fingerprints -> film_fingerprint`
   - 输入：所有 scene fingerprints，按 scene 顺序排列
   - 输出：`fingerprints/film_fingerprint.json`
   - 目标：得到全片的语义轨迹和主导表达模式

3. `scenes + film_fingerprint -> film_analysis`
   - 输入：`scenes.json` 和 `film_fingerprint`
   - 输出：`analysis/film_analysis.json`
   - 目标：生成高密度全片分析，并受 fingerprint 约束

4. `scenes + film_analysis + fingerprints -> experience_cards`
   - 输出：`analysis/experience_cards.jsonl`
   - 每张 card 增加 `fingerprint`
   - 目标：让 card 既有解释密度，也能被稳定检索

5. `associate input -> query_fingerprint`
   - 输出：`outputs/key_associates/...json` 中增加 query fingerprint
   - 用 query fingerprint 匹配 card fingerprints
   - 再把召回 cards 作为脚本创意素材

## Test Plan

- Schema tests：
  - fingerprint 字段存在且类型正确
  - `confidence` 在 0..1
  - evidence 至少包含 `scene_id` 和引用字段
- Unit tests：
  - scene fingerprint 可从 mock `SceneAnalysis` 生成
  - film fingerprint 可按 scene 顺序聚合
  - experience card 能携带 fingerprint 并通过 validation
- Retrieval tests：
  - query fingerprint 能召回包含相近 `interaction_mode` / `audience_projection` 的 cards
  - 腾讯“屏幕后对话型” brief 不应只召回故事化、艺术片式 cards
- Regression tests：
  - 原有 `mock-run`
  - 原有 `analyze-scenes`
  - 原有 `associate`

## Assumptions

- 第一版采用“半受控标签”：固定维度，标签值由 LLM 生成并做归一化。
- fingerprint 是独立中间产物，不嵌入同一个大 prompt 一次性生成。
- 第一版先做 JSON 文件检索，不引入 vector DB。
- `film_analysis` 和 `experience_cards` 的生成都应读取 fingerprint，但不负责重新定义 fingerprint。
