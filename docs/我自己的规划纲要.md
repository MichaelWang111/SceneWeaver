# 1.需要做一个fingerprint system，

现在film_analysis和key_associates都已经完善了，能够获得例如:
  - why_it_works
  - audience_projection
  - underlying_emotion
这样的High-density creative reasoning。
但是缺少了latent expression/tag这一层也就是低维语义坐标
例如：
emotion_core:
  - awakening
  - ambition

audience_projection:
  - future_builder

visual_motifs:
  - silhouette
  - sunset
  - upward_motion

symbolism:
  - becoming
  - destiny

pace:
  - explosive_build

而这个东西才是runtime retrieval的关键。
设想中，同时生成latent expression/tag和High-density creative reasoning,然后其实是将keys的associate latent expression和film_analysis的latent expression做匹配。
所以在这种情况下 高维表示和低维表示的关系需要做一个比较有构建性的一个关联，看看他们之间是互为约束还是互为补充。

# 2.latent 匹配与runtime retrieval

第二步，将film_analysis的latent expression按时间顺序整理来直接表示这个film,生成film_analysis和experience_cards，input key的latent expression就可以与experience_cards做语义匹配检索了。
这样才能做到runtime retrieval。
