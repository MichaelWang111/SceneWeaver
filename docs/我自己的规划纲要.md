# 当前规划纲要

## 1. 核心判断

SceneWeaver 当前不应该继续扩张中间 schema，而应该把核心解析架构压缩为：

```text
标签层
经验卡片层
```

也就是：

```text
SceneAnalysis.tags
ExperienceCard.tags
```

## 2. tags 的定义

这里的 tags 不是开放式关键词，而是半封闭标签词表：

```text
canonical tags
aliases
candidate pool
deprecated mappings
```

正式数据只写 canonical tags。模型或人工分析中出现的新说法，先进入 candidate pool，再判断是合并 alias、升级为新 tag，还是丢弃。

## 3. 为什么这样做

旧思路把低维语义坐标做成独立 fingerprint 中间层，容易继续膨胀出 scene wrapper、film wrapper、query wrapper 和 card wrapper。

新思路更直接：

```text
analysis 是视频解析主产物
tags 是 analysis 的可检索索引
cards 是可复用经验单元
```

## 4. 当前执行方向

第一步：

```text
修改 schema 为 tags + experience cards
```

第二步：

```text
升级标签词表的迭代管理
```

## 5. 长期目标

当用户输入一个粗糙 brief 时，系统应能：

1. 生成 query_tags。
2. 召回相关 experience cards。
3. 用真实视频经验补足自由联想的盲区。
4. 为导演稿生成提供可落地的策略、技法和避坑建议。
