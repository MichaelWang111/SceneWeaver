# 复盘检索样本

这个目录存放“全片复盘”后的检索原型样本。它和正式生产输出目录 `outputs/film_analysis` 是分开的，不遵循当前生产输出布局。

`retrieval_review` 表示第二轮复盘：第一轮观看观察已经存在，模型也已经看完整个视频，因此可以结合上下文重新理解每个 scene 的功能、脚本用途和可复用导演经验。

## 数据层

- `source.json`：数据源层，包括 fake frames 和 scene package。
- `knowledge.json`：全片复盘后的高阶知识，比如拍摄手法、导演意图、情绪功能和复用风险。
- `retrieval.json`：面向检索的数据，包括宽松 LLM 标签、一句话脚本用途和占位 embedding 向量。

## 当前样本规模

- `collection_manifest.json`：完整样本集合索引。
- 一共 16 个视频：原 5 个主流行业样本，加 1 个半导体/先进制造扩展样本。
- 每个视频 10 个 scene，总共 160 条 retrieval item。

## 行业

- 医疗科技与数字健康
- 能源与公共事业
- 汽车与出行
- 金融服务与保险
- 消费零售与供应链
- 半导体与先进制造

## 风格

- 商务企业品牌片
- 纪录片式企业品牌片
- 故事型企业品牌片
- 世界 500 强质感的竖屏短片

## 说明

`enterprise_healthtech_story_001` 是最早手写的样本。其他样本是由 `generate_fixtures.py` 根据结构化行业和风格设定生成的。它们仍然是假数据，但目的是支持跨行业、跨风格、跨脚本阶段、跨创作目的、宽松 LLM 标签和未来 embedding 字段的检索原型测试。

## 检索规则

默认用这一层作为检索标签和经验搜索的数据来源。`retrieval_viewing` 只作为验证和审计来源保留，不作为最终检索目标。

## 命名说明

这个目录替代旧的 `retrieval_prototype` 名称。旧数据被保留下来，因为它适合作为全片复盘目标和验证样本。
